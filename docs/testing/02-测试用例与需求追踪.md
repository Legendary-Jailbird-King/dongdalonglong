## 2. 核心手工验收用例

| 用例编号 | 场景 | 前置条件 | 步骤 | 预期 |
|---|---|---|---|---|
| UAT-001 | 三角色业务闭环 | 三角色账号、商家有菜品、顾客有地址 | 顾客下单→商家接单→骑手抢单→确认送达 | 状态依次为 0→1→2→3，骑手最终空闲 |
| UAT-002 | 顾客取消订单 | 待处理订单 | 顾客点击取消 | 状态变为 4，商家收到通知 |
| UAT-003 | 商家拒单 | 待处理订单 | 商家点击拒单 | 状态变为 5，不能再接单或抢单 |
| UAT-004 | 菜品下架/删除 | 商家有在售菜品 | 下架或删除后顾客刷新 | 顾客不可见且不可下单 |
| UAT-005 | 越权访问 | 使用商家 Token | 请求顾客地址接口 | 返回 403，不泄露数据 |
| UAT-006 | 并发抢单 | 两个空闲骑手、一个已接订单 | 同时点击抢单 | 仅一个成功，另一个明确失败 |

## 3. 执行命令

```powershell
# 首次安装
cd backend
python -m pip install -r requirements-test.txt
cd ../frontend
npm install

# 全套执行并生成日志和覆盖率
$env:PYTHON = "Python 可执行文件路径"
.\run_tests.ps1
```
