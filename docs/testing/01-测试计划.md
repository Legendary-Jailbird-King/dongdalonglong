# 全栈外卖管理系统测试计划

## 1. 文档控制

| 项目 | 内容 |
|---|---|
| 文档编号 | TMS-TP-001 |
| 被测版本 | Git commit `20f826f` |
| 测试类型 | 单元、接口、集成、契约、构建、并发风险测试 |
| 测试基准日 | 2026-06-13 |

## 2. 测试目标

验证认证与角色隔离、菜品和地址管理、下单事务、订单状态机、骑手分配、WebSocket 鉴权以及前端构建满足项目计划书和 README 描述的业务契约；发现影响业务闭环、数据一致性与安全性的缺陷。

## 3. 范围

### 3.1 范围内

- C1 认证：注册、登录、JWT、RBAC、非法输入。
- C2 顾客：商家/菜品浏览、地址软删除、下单、历史订单、取消。
- C3 商家：菜品 CRUD、接单、拒单、指派骑手。
- C4 骑手：可抢订单、抢单、我的订单、确认送达。
- C5 数据层：金额计算、事务回滚、软删除过滤、并发抢单风险。
- WebSocket：连接鉴权。
- 前端：API 契约、路由守卫、认证存储、生产构建。

### 3.2 范围外

- MySQL 兼容性、生产部署、真实弱网和浏览器兼容性。
- 性能容量、渗透测试、第三方基础设施故障恢复。
- UI 视觉验收和移动端真机测试。

## 4. 测试策略

| 层级 | 方法 | 工具 | 退出标准 |
|---|---|---|---|
| 单元 | 密码/JWT、状态映射 | pytest | 全部通过 |
| API/集成 | 黑盒接口 + 隔离 SQLite | FastAPI TestClient、pytest | P0/P1 用例通过 |
| 并发 | 双骑手同时抢单 | ThreadPoolExecutor、pytest | 仅一个请求成功 |
| WebSocket | 合法/非法连接 | TestClient WebSocket | 鉴权符合预期 |
| 前端契约 | 静态业务契约 | Node test runner | 全部通过 |
| 构建 | 生产打包 | Vite | 构建成功 |

## 5. 环境与数据

- 操作系统：Windows，PowerShell。
- 后端：Python 3.10+，测试库 `backend/tests/test_takeaway.db`。
- 前端：Node.js 22、npm 10。
- 测试数据：每条后端用例执行前清空测试库，禁止访问开发库 `takeaway.db`。
- 证据：`test-artifacts/logs/` 原始日志和 `test-artifacts/coverage/` 覆盖率报告。

## 6. 准入与退出标准

准入：源码可克隆、依赖可安装、后端可导入、前端包清单完整。

退出：

- 自动化测试完成执行并保留日志。
- P0/P1 缺陷有明确记录；未修复项必须写明风险。
- 测试用例、需求追踪、执行报告齐全。
- 前端生产构建成功。

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| SQLite 与 MySQL 锁语义不同 | 并发结论不完全等价 | MySQL 上补做并发回归 |
| 无正式需求规格说明书 | 预期可能存在歧义 | 以计划书、README、API 实现三方交叉确认 |
| 缺少浏览器 E2E | UI 流程风险残留 | 后续补充 Playwright 三角色闭环 |
